# API: Resource & Endpoint Workflow

> Read this when adding or modifying API resources or oRPC endpoints.

---

## Resource Structure

Every resource lives in `apps/api/src/resources/<name>/`:

```
<name>/
├── <name>.schema.ts       # Drizzle table definition (pgTable)
├── endpoints/             # One file per oRPC endpoint (filesystem-routed)
│   ├── list.ts
│   ├── create.ts
│   ├── [paramId]/
│   │   ├── get.ts
│   │   ├── update.ts
│   │   └── delete.ts
│   └── nested-group/      # Non-param subdirs = nested router groups (camelCased)
│       └── action.post.ts
└── handlers/              # Optional: event-bus side-effect handlers
    └── on-created.ts
```

No `index.ts` barrels needed — codegen discovers endpoints from the filesystem.

## Filesystem Routing Conventions

The router is auto-generated by `scripts/codegen-router.ts`. Run after any file add/remove:

```bash
cd apps/api && npx tsx scripts/codegen-router.ts
```

**File naming → HTTP method:**
| Filename | Method | Path segment |
|----------|--------|-------------|
| `list.ts` | GET | (none — collection root) |
| `create.ts` | POST | (none) |
| `get.ts` | GET | (none) |
| `update.ts` | PUT | (none) |
| `delete.ts` | DELETE | (none) |
| `name.get.ts` | GET | `/name` |
| `name.post.ts` | POST | `/name` |

**Directories:**
- `[param]/` → `{param}` dynamic segment in URL
- `non-param-dir/` → nested router group (camelCased key)

**Examples:**
```
users/endpoints/list.ts                  → GET  /users             → orpcClient.users.list
users/endpoints/[userId]/get.ts          → GET  /users/{userId}    → orpcClient.users.get
users/endpoints/[userId]/update.ts       → PUT  /users/{userId}    → orpcClient.users.update
```

## Add New Endpoint

1. Create file in `resources/<name>/endpoints/` following naming conventions above
2. Default-export an endpoint:
   ```typescript
   import { z } from 'zod';
   import { isAuthorized } from '@/procedures';

   export default isAuthorized
     .input(z.object({ ... }))
     .output(z.object({ ... }))
     .handler(async ({ input, context }) => { ... });
   ```
3. Run codegen: `npx tsx scripts/codegen-router.ts`
4. Verify: `pnpm --filter api tsc --noEmit`

## Endpoint Invariants

### Auth
- `isPublic` — no auth required
- `isAuthorized` — requires user. Provides `context.user` typed as `User`

### Schemas
- Always provide `.output()` — required for type inference to web
- Use constants from `app-constants` for enums: `z.enum(STATUSES)`, not `z.enum(['ready', 'failed'])`
- For Zod tuples used across schemas, define once in constants

### Imports
- Use `@/` alias for all imports: `@/db`, `@/procedures`, `@/services/...`
- Never use relative imports — `tsconfig.baseUrl` is `src` with `@/` path alias

### Service helpers
- Put complex external-API logic in `src/services/` — keep endpoints thin

## Database (Drizzle + Postgres)

`db.<table>` provides a `DbService` wrapper:

```typescript
import db from '@/db';

// Query
const user = await db.users.findFirst({ where: { id: userId } });
const users = await db.users.find({ where: { status: 'active' }, orderBy: { createdAt: 'desc' } });

// Mutations
await db.users.insertOne({ email, fullName });
await db.users.updateOne({ id: userId }, { status: 'active' });
await db.users.deleteOne({ id: userId });
```

Filter operators: `{ field: value }` for equality, `{ field: { gt, gte, lt, lte, in, notIn, like, ilike, isNull, isNotNull } }`, `OR: [...]`, `AND: [...]`.

## `shouldExist` Middleware

oRPC middleware that loads an entity by `input.id` and throws NOT_FOUND if missing. Defined in `src/procedures.ts`.

```typescript
import { shouldExist, isAuthorized } from '@/procedures';
import db from '@/db';

export default isAuthorized
  .input(z.object({ id: z.string(), data: ... }))
  .use(shouldExist((id) => db.users.findFirst({ where: { id } }), 'User'))
  .output(outputSchema)
  .handler(async ({ input, context }) => {
    return db.users.updateOne({ id: input.id }, input.data);
  });
```
